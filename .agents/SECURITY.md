# 보안 수칙 및 체크리스트 (Security)

이 문서는 프로젝트 개발 시 준수해야 할 최소한의 보안 요구사항을 정의합니다.

## 1. 코드 보안 기본 수칙
- **비밀키 관리**: 로컬 개발이든 서버든 환경변수 파일(`.env`)에 포함된 토큰이나 비밀번호가 저장소에 커밋되지 않도록 확인합니다.
- **의존성(Dependency) 검사**: 사용 중인 라이브러리의 취약점을 주기적으로 검사 및 업데이트합니다.

## 2. 데이터 보호 정책
- **저장소 데이터**: PII(개인 식별 정보)는 적절히 암호화하여 저장합니다.
- **전송 데이터**: API 통신 등 모든 전송 데이터는 HTTPS 프로토콜을 사용해야 합니다.

## 3. 취약점 방어 (OWASP Top 10)
- **SQL Injection 방어**: 모든 쿼리는 파라미터 바인딩(ORM 활용)을 사용해야 합니다.
- **XSS/CSRF 방어**: 프론트엔드/백엔드 아키텍처 상 검증된 방어 기법을 활성화합니다.
