# 시스템 안정성 (Reliability)

프로덕션 장애를 최소화하고 신속히 복구하기 위한 최소 기준이다. 배포 파이프라인 상세는 `workflows/devops.md`를 따른다.

## 1. 복원력 (Resilience)

**필수 패턴**
- [ ] 외부 호출: 타임아웃(기본 10s) + 지수 백오프 재시도(최대 3회)
- [ ] 외부 장애 전파 차단: Circuit Breaker
- [ ] DB 쓰기: 트랜잭션 + 롤백 처리
- [ ] 백그라운드 작업(Celery): 실패 시 Dead Letter Queue

**설계 원칙**: Graceful Degradation(부분 장애 시 핵심 기능 유지) · Fail Fast(조기 감지) · Idempotency(재시도 안전).

## 2. 로깅·모니터링

| 레벨 | 용도 |
|------|------|
| DEBUG | 개발 시 상세 추적(프로덕션 비활성화) |
| INFO | 정상 비즈니스 이벤트 |
| WARNING | 정상 범위 내 이상(재시도, 느린 쿼리) |
| ERROR | 복구 가능한 오류(API 실패, DB 타임아웃) |
| CRITICAL | 즉시 대응(데이터 손상, 시스템 다운) |

**필수사항**
- [ ] ERROR 이상: 스택 트레이스 + 컨텍스트(요청 ID·사용자 ID) 포함
- [ ] 민감 정보(비밀번호·토큰) 로그 금지
- [ ] 구조화 로깅(`structlog` / JSON) 권장
- [ ] `/health` 엔드포인트 및 메트릭 엔드포인트(Prometheus 호환) 제공

## 3. 무중단 배포·백업

- [ ] 블루-그린 또는 롤링 업데이트 적용
- [ ] 배포 전 롤백 계획 수립, 배포 후 5분 이내 smoke test
- [ ] DB 마이그레이션은 하위 호환(컬럼 삭제는 2단계 배포)
- [ ] DB 자동 백업(≥일 1회) + 분기 1회 복원 테스트

## 4. 장애 대응 (Incident Response)

| 등급 | 정의 | 대응 시간 |
|------|------|-----------|
| P1 | 서비스 전체 중단 | 즉시(≤15분) |
| P2 | 주요 기능 장애 | ≤1시간 |
| P3 | 부분 기능 이상 | ≤4시간 |
| P4 | 경미 이슈 | 다음 스프린트 |

**절차**: 감지 → 등급 판정 → 롤백/핫픽스 → smoke test & 로그 확인 → 포스트모템(`docs/evaluations/qa-reports/`에 기록).
