# AI Harness Metrics (자가 개선 지표)

<!-- AI Harness Rule: Definition of objective metrics to evaluate the harness health. -->

> 프로젝트 메인 관리 하네스(`.agents/`) 자체가 효율적이고 효과적으로 동작하는지, 낡은 문서나 불필요한 관행은 없는지 지속 측정하기 위한 **객관적 판단 지표**입니다. `/improve` 워크플로우에 의해 이 지표가 수집/평가됩니다.

---

## 1. Handover(태스크 이관) 효용성

| 평가지표 | 정의 및 측정 방법 | 목표 값 (Threshold) |
| -------- | ---------------- | ------------------- |
| **핑퐁 횟수(Ping-Pong)** | 특정 태스크를 위해 역할 이관이 반복 체류하는 횟수 (보통 1기능 당 BE→FE→QA 로 끝나야 정상. QA→FE 등 역방향 회귀가 잦으면 불량) | 기능 단위당 **최대 왕복 1회** 이하 |
| **핸드오버 완료 소요율** | 생성된 `to_*.md` 로그 중 명시된 기한 내 완료 처리되어 `logs/` 로 이동한 비율 | 스프린트 내 **90% 이상** |

*문제 발생 시 (권고 조치)*: 
- 핑퐁이 잦을 경우, 워크플로우 스텝 조정 또는 Handover 양식 내 필수 요구 정보 누락 개선.

---

## 2. 컨텍스트 및 원칙(Rule) 준수율

| 평가지표 | 정의 및 측정 방법 | 목표 값 (Threshold) |
| -------- | ---------------- | ------------------- |
| **린터/컨벤션 위반율** | PR 제출 또는 자동 점검 시 `.agents/rules/`의 내용 위반으로 차단되거나 검증 스킬(`verify-*`)에서 발견된 위반 건수 | PR 당 **0건** 유지 |
| **하네스 문서 참조율** | 에이전트 작업 중 `ARCHITECTURE.md` 등 코어 시스템 기준을 무시하고 수정해버리는 빈도 | **절대 금지 (0건)** |
| **컨텍스트 표류(Context Drift)**| 각 `contexts/*.md` 파일이 코드의 최신 상태와 맞지 않아 업데이트가 지연된 비율 | 작업 종료 후 **항상 100% 동기화** |

*문제 발생 시 (권고 조치)*:
- 기계적 린터 룰(`linter-rules.md`) 보강. 
- 복잡해서 지키지 않는 아키텍처 문서는 현실에 맞게 리팩토링/최적화.

---

## 3. 에이전트 도구 및 워크플로우 효율성

| 평가지표 | 정의 및 측정 방법 | 목표 값 (Threshold) |
| -------- | ---------------- | ------------------- |
| **스킬(Skill) 정상 실행 비율** | 명세된 스킬을 호출할 때 Syntax 에러, 모델 컨텍스트 초과 등으로 중단되지 않고 정상 동작(종료)된 비율 | **95% 이상** |
| **슬래시 커맨드 롤백 비율** | `/backend` 등 워크플로우 실행 실패로 인해 롤백 절차가 작동한 비율 | **10% 미만** |

*문제 발생 시 (권고 조치)*:
- 자주 실패하는 스킬은 `skills/` 코드를 개선하거나 모델 인자 최적화 필요.
- 워크플로우가 모호하다면 해당 직군 워크플로우 명세(예: `workflows/backend.md`)의 Steps를 더욱 잘게 쪼개거나 구체화.

---

## Harness Health Score (종합 평가 산출)

평가 스킬(`evaluate-harness`)은 위의 3개 축을 분석하여 **100점 만점의 Health Score**를 산출합니다.

- **A (90~100점)**: 우수함. 하네스 규칙과 스킬 유지 유지방안이 원활함.
- **B (70~89점)**: 정상. 일부 문서 노후화나 잦은 핑퐁이 발생하고 있으나 감내할 수준. (작은 개선만 권고)
- **C (50~69점)**: 개선 필요. 자동 오류나 워크플로우 실패가 잦아 명세나 린터 전반의 리팩토링 검토 필요.
- **D (50점 미만)**: 위험. 하네스 체계가 무너져 생산성을 떨어뜨리고 있으므로, `Architect`와 `Master`급의 즉각적 하네스 전면 조정 요구됨.
