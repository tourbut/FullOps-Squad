# 핵심 기술 신념 (Core Beliefs)

우리가 기술적인 결정을 내릴 때 기준으로 삼아야 할 신념들입니다.

1. **가독성(Readability) > 작성 편의성(Writability)**
   - 나중에 코드를 유지보수할 다른 개발자(또는 에이전트)를 위해 코드를 명확히 작성합니다.
   
2. **명백한 상태 관리**
   - 어플리케이션의 상태 변화는 추적 가능해야 하며 숨겨진(Side effect) 동작을 지양합니다.
   
3. **작고 빠른(Iterative) 변경**
   - PR이나 실행 계획(Execution Plan)은 가능한 작게 나누어 즉시 피드백을 받도록 합니다.
