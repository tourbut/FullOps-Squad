# 코드 품질 및 검토 기준 (Quality Score)

이 문서는 코드 베이스의 상태와 건강도를 측정하기 위한 프레임워크입니다.
테스트 및 품질 보증(QA)에 관련된 추가 규칙은 `rules/unit-testing.md`, `rules/integration-testing.md`, `skills/qa-tester`를 함께 참고하십시오.

## 1. 코드 리뷰 체크리스트
- **가독성**: 변수명, 함수명은 직관적인가?
- **의존성**: 순환 참조나 불필요한 결합은 없는가?
- **에러 핸들링**: 모든 엣지 케이스에 대한 에러 처리가 되어 있는가?
- **성능**: 시간이나 공간 복잡도상 불필요한 병목은 없는가?

## 2. 테스트 커버리지 요구사항
- 새로 추가되는 로직에 대해 최소 N% 이상의 단위 테스트 커버리지를 요구합니다.
- CI 파이프라인 상에서 자동으로 테스트 및 린팅이 통과되어야 합니다.

## 3. 코드 냄새(Code Smells) 탐지
- 너무 긴 함수/클래스
- 중복된 코드 덩어리
- 과도한 매개변수
